# SQL tuning methodology (practical checklist)

这份方法论用于补充案例与算子分析，帮助形成“先定位、再改写、后结构、最后参数”的稳定节奏。

## 1) 四阶段方法

1. **定位问题**：先确认慢在扫描、传输、JOIN、聚合、排序中的哪一类。
2. **改写 SQL**：优先做低风险 rewrite（前移过滤、去 `SELECT *`、修复隐式转换）。
3. **结构优化**：再评估索引、分布键、分区设计、存储方式。
4. **参数验证**：最后才考虑 GUC/会话参数验证，不作为第一优先级。

## 2) JOIN 分析最小清单

- JOIN 键两侧类型是否一致；
- JOIN 前是否已经尽早过滤；
- JOIN 键是否与分布键匹配（避免 `REDISTRIBUTE`）；
- 是否存在可下推但未下推的条件。

## 3) 聚合/排序分析最小清单

- 是否可以先局部聚合再全局汇总；
- `DISTINCT` 是否可等价改写；
- 排序是否仅为展示需求，可否后移或减少输入列。

## 4) 验证与回归

- 先 `EXPLAIN*` 验证计划变化；
- 再看总耗时、CPU、扫描行数、网络传输量；
- 必须保留回滚路径（恢复原 SQL/索引/参数）。
